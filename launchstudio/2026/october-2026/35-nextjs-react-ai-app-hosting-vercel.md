# How to Host Apps After Using AI To Code
If you are an AI-native founder, you have likely spent the last few weeks in a sandbox. Tools like Bolt.new, Lovable, or v0 allow you to type prompts and watch a fully functional Next.js or React interface materialize before your eyes. 

These sandbox environments are magical for prototyping. But eventually, you have to launch. You cannot sell a B2B SaaS to an enterprise client while your app is hosted on a temporary URL like `bolt-project-xyz123.web.app`. 

To turn your prototype into a real business, you must export the code and host it on a professional infrastructure platform like Vercel. However, transitioning an AI-generated Next.js app from a sandbox to a live production server is rarely as simple as clicking an "Export" button. Here is what you need to know about hosting your AI app, and why professional deployment is critical.

## Why Vercel is the Standard for AI Apps

When AI code generators write frontend code, they overwhelmingly choose **Next.js** (a React framework). Next.js was created by a company called **Vercel**. Consequently, Vercel is the absolute best place to host an AI-generated Next.js application.

### 1. The Edge Network
Unlike traditional hosting (where your app lives on one server in one city), Vercel deploys your frontend to an "Edge Network." This means copies of your app are distributed to servers all over the world. When a client in Amsterdam loads your app, they connect to an Amsterdam server, resulting in lightning-fast loading speeds.

### 2. Serverless API Routes
AI applications rely heavily on API calls (sending prompts to OpenAI and waiting for responses). Vercel provides "Serverless Functions." This allows your Next.js app to execute secure, backend API calls without you having to set up and maintain a dedicated Node.js server. 

### 3. Continuous Deployment (CI/CD)
When you host on Vercel, it connects directly to your GitHub repository. This means every time you generate a new feature in your AI tool and push the code, Vercel automatically detects the change, rebuilds your app, and updates your live website with zero downtime.

## The Deployment Trap for Non-Technical Founders

While Vercel is powerful, getting an AI-generated sandbox app onto Vercel is highly technical. 

When you export code from an AI builder, it is often incomplete. The AI assumes you know how to configure `.env` (environment variable) files to hide your OpenAI API keys. It assumes you know how to set up GitHub repositories. It assumes you understand how to configure Cross-Origin Resource Sharing (CORS) policies so that your new custom domain can actually talk to your Supabase database.

If you skip these steps, one of two things happens:
1. Your Vercel deployment simply crashes with a "Build Error" that you do not know how to fix.
2. Your app goes live, but your API keys are exposed to the public internet, leading to a massive financial theft of your AI credits.

## Frequently Asked Questions

### Can I just keep my app hosted on Bolt.new or Lovable?
For testing, yes. For a real business, no. Sandbox hosting platforms are not designed for high-traffic production use. Furthermore, B2B clients will not trust or pay for software hosted on a temporary `.web.app` subdomain. You need a custom domain.

### Do I have to pay for Vercel?
Vercel has a very generous free tier ("Hobby") which is perfect for testing. However, once you start charging users for a commercial SaaS, Vercel's Terms of Service require you to upgrade to their "Pro" tier, which costs $20 per month per developer. 

### What is an Environment Variable (`.env`)?
An environment variable is a secure way to store sensitive information (like your OpenAI API key or Stripe Secret Key) on your server. If you hardcode these keys directly into your React files, they will be stolen. Vercel allows you to inject these keys securely during the build process.

### Why do I need GitHub to host on Vercel?
While you can deploy directly from your computer via a command line, linking Vercel to GitHub is the industry standard. It creates a CI/CD pipeline: every time you (or an AI tool) push an update to your GitHub code, Vercel automatically detects the change and updates the live website.

### How does LaunchStudio help with future updates?
When we deploy your app, we set up the GitHub-to-Vercel pipeline. If you want to use an AI builder to design a new dashboard later, you can simply commit the new frontend code to GitHub. Vercel will automatically update your live site without you needing to pay us for another deployment.
