# Social Media Post: Infrastructure as Code (IaC) for AI Startups: Terraform vs. Pulumi

"ClickOps" (manually clicking buttons in the Vercel or Supabase dashboards to change settings) is fine for a hackathon. 

It is a massive security and SOC 2 compliance risk for a funded AI startup. If an engineer accidentally deletes an environment variable or makes a Storage bucket public, your app crashes or leaks data.

You need Infrastructure as Code (IaC). But should you use Terraform or Pulumi?

Terraform (HCL) is the industry standard. Every DevOps engineer knows it. But it uses a proprietary configuration language.

Pulumi lets you write infrastructure in TypeScript. For a Next.js/Node AI startup, this means your full-stack devs can manage infrastructure using the exact same language, IDE, and linting tools they already use.
