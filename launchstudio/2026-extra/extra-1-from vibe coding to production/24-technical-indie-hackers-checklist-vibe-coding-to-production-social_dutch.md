Een checklist voor technische founders die hun eigen terminalcommando's kunnen draaien. Elk item is een TEST, geen overtuiging. 🧠

❌ "Mijn eigen code lezen en beoordelen of het correct oogt" bevestigt wat je al gelooft
❌ Jouw eigen autorisatielogica reviewen heeft een structurele blinde vlek die geen vaardigheid wegneemt
❌ Een pipeline die "bestaat en draait" ≠ een pipeline die daadwerkelijk slechte merges blokkeert
❌ Sequentieel solo testen kan structureel geen gelijktijdigheidsbugs naar boven brengen

✅ git log --all --full-history voor geheimen — volledige geschiedenis, niet alleen huidige bestanden
✅ Roep jouw API direct aan, de frontend omzeilend — geeft het een 403 terug of de data?
✅ Vuur twee gelijktijdige verzoeken af op een gedeelde resource — slaagt precies één?
✅ Herstart het serverproces (niet alleen verversen) — overleeft de data?
