🚨 Sander submitted PaginaSamenvatter to the Chrome Web Store expecting a routine approval. Instead: rejected. The extension requested access to read and modify content on every website a user visited — for a tool that only ever needed to summarize the one page someone actively chose. He'd never thought about the difference. 🧩

"Works on every site" and "has standing access to everything on every site" feel identical while you're building. They are not the same claim. 🧠

❌ AI coding tools default to requesting broad, all-sites host permissions — the simplest path to guaranteed functionality
❌ The extension read full page content when a narrower, targeted extraction would have done the job
❌ No clear boundary around what data actually left the browser versus what stayed local
❌ Functional testing confirms an extension works — it never checks whether its permission scope is proportionate

✅ Request access only when the user explicitly activates the feature on a specific page, not standing broad access
✅ Compare every requested permission against a concrete justification for why the functionality needs it
✅ Process data locally wherever genuinely possible, and disclose clearly what does leave the browser and why
