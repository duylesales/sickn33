## Cấu trúc tuần mẫu (Weekly Framework)

| Day | Theme | Goal | Format |
|:---|:---|:---|:---|
| **Monday** | Educational / Deep Dive | Establish authority | Long thread (X) / Article (LI) |
| **Tuesday** | Visual / Transformation | Show capability | Before/After image or short video |
| **Wednesday** | Quick Tip / Technical | Value add | Short post with code snippet |
| **Thursday** | Content Promotion | Drive blog traffic | Hook + Link to recent article |
| **Friday** | Social Proof | Build trust | Client quote / Case study snippet |
| **Saturday** | Hot Take / Opinion | Engagement | Provocative question or statement |
| **Sunday** | Resource | Lead Gen | Checklist or Template giveaway |

### 4. The "Educational Thread" (Authority)
**Twitter/X:**
🧵 5 reasons your AI prototype will fail its first real user test (and how to fix them):

1/ The "Happy Path" delusion.
AI writes code that works perfectly... when everything goes right. But what happens when Stripe declines a card? Does your app crash? You need robust error handling.

2/ The Loading State.
AI loves to assume instant API responses. Real-world LLM calls take 3-10 seconds. If you don't have skeleton loaders or streaming UI, users will think your app is broken.

3/ Database Chaos.
Without proper relational structure and RLS, your users will start seeing each other's data as soon as you have more than 1 person logged in.

4/ Environment Variables.
"It works on my machine!" — Yes, because your keys are local. Moving from localhost to a secure production deployment requires proper CI/CD pipelines.
