🚑 **Adding more developers to a late software project mathematically makes it later.**

A CEO is furious that an offshore **software project** is 3 months overdue. They double the budget and force the agency to add 5 new developers to speed it up.
Two months later, the project is completely dead. 

This is Brooks's Law (1975). Adding people to a fragile, late project forces senior devs to stop coding to train juniors. The juniors don't understand the fragile codebase, so they introduce catastrophic bugs, halting all progress.

A late project is not a capacity problem. It is an architectural collapse (Technical Debt). 
Standard agencies will suggest a "Big Bang Rewrite" (trashing the code and starting over). Do not accept this. It is business suicide.

Elite teams execute a **Strangler Fig Migration**. You put an API Gateway in front of the dying app, and systematically replace broken features with new microservices one-by-one, keeping the business operational.
