⚠️ **When NOT to use React Native: A CTO’s Architectural Guide.**

React Native is incredible for 90% of B2B SaaS apps, slashing development costs by 40%. But if you choose it for the wrong use case, you will waste millions.

When cross-platform development fails:
🛑 **Heavy 3D or AR/VR:** The JavaScript layer cannot handle complex real-time rendering. The app will drop frames and overheat. Use native Metal/Vulkan or Unity.
🛑 **Complex IoT and Bluetooth:** Relying on buggy open-source libraries for mission-critical Bluetooth connections is a liability. You need deterministic, pure Swift/Kotlin code.
🛑 **Bleeding-Edge "OS-First" Features:** If your app relies on integrating Apple's newest hardware API announced yesterday, you can't wait months for the React Native community to build a bridge.
