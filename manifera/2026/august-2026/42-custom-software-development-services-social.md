When evaluating **custom software development services**, marketing slide decks are a lagging indicator of quality. The only leading indicator is how they build code when the client isn't looking.

Here is the 4-step technical due diligence framework for CTOs:

1️⃣ **Audit the Git Commits:** Are they pushing small, atomic changes daily? Or pushing 4,000 lines at 11:30 PM on a Friday? Massive commits = no code review.
2️⃣ **Audit the PR Hygiene:** Did the senior engineer leave architectural feedback, or just type "LGTM" and hit approve? "LGTM" is how technical debt gets merged.
3️⃣ **Audit the CI/CD Pipeline:** If human developers can bypass automated unit tests, the tests do not exist.
4️⃣ **Audit Database Evolution:** Are they using version-controlled migrations (Flyway/Prisma), or running manual SQL scripts on production like cowboys?
