📱 "We need a mobile app."

The question that shapes your budget for the next 12-18 months.

**3 options. Very different costs:**

**Option 1: Native (Swift + Kotlin)**
💰 €80K-€160K for both platforms
👥 2-4 mobile developers needed
⏰ 4-8 months to first release
✅ Best for: games, AR/VR, GPU-intensive apps

**Option 2: Cross-Platform (React Native / Flutter)**
💰 €40K-€80K (40-60% cheaper than native)
👥 2-3 developers needed
⏰ 2-4 months to first release
✅ Best for: 95% of B2B apps (dashboards, CRMs, tools)

**Option 3: PWA (Progressive Web App)**
💰 <€40K
👥 Existing web team
⏰ 1-2 months to first release
✅ Best for: content apps, low-frequency use, rapid iteration

**React Native vs Flutter in 2026:**

React Native:
→ TypeScript (massive talent pool)
→ Code sharing with React web apps (30-40% reuse)
→ Largest library ecosystem

Flutter:
→ Better animation performance
→ Single codebase for mobile, web, desktop
→ Slightly faster hot reload

**The 2026 default recommendation:**

1. Start with a responsive web app
2. If mobile data shows users want native → React Native or Flutter
3. Go native only if you hit specific technical limitations

**Maintenance reality:** Budget **15-25% of initial cost annually.**

€60K app = €9K-€15K/year for OS updates, patches, and App Store compliance.

Skip maintenance → app crashes on new iOS/Android → App Store removal.

Don't build mobile-first. Build evidence-first.
