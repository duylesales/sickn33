💰 CEO budgets €120K for a custom app.

18 months later: total spend = €300,000.

The build was only **40% of the total cost.**

The other 60%? Maintenance.

This is not an exception. It's the rule.

Research: maintenance = **60-80% of total cost of ownership.**

**The 4 types of maintenance:**

🔧 **Corrective** (20-25%) — Bug fixes from real-world usage
🔄 **Adaptive** (25-30%) — iOS updates, API changes, framework deprecations
✨ **Perfective** (30-35%) — New features users actually want
🛡️ **Preventive** (10-15%) — Refactoring, dependency updates, monitoring

**The maintenance budget formula:**

📊 Annual maintenance = **15-25% of original build cost**

€100K build → €15K-€25K/year maintenance
5-year TCO: **€175K-€225K** (not €100K)

Year 1 is most expensive (25-35%). Years 2-5 stabilise at 15-20%.

**5 things that INFLATE maintenance costs:**

❌ No automated tests → 3x longer bug fixes
❌ No documentation → new developers reverse-engineer everything
❌ Trendy framework that dies → nobody can work on it
❌ Single developer holds all knowledge → departure = crisis
❌ Technical debt → 30-min hack becomes 10-hour investigation

**5 things that REDUCE maintenance costs:**

✅ Test suite (80%+ coverage) → 30-40% fewer bugs
✅ CI/CD pipeline → 10-min deploys instead of 4-hour ordeals
✅ Logging + monitoring → issues detected in minutes
✅ Architecture docs → new devs productive in weeks, not months
✅ Dependency policy → prevents 3-year-old upgrade nightmares

**The retainer model:**

Basic: 10-20 hrs/month (€2K-€4K) — patches + bug fixes
Standard: 20-40 hrs/month (€4K-€8K) — above + small features
Premium: 40-80 hrs/month (€8K-€16K) — above + major features

Predictable costs. Guaranteed availability. Institutional knowledge.

Software isn't a building. It's a garden. Stop maintaining it and it dies.
